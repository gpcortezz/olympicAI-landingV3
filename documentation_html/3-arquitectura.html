3.1. Diseño General

OlympicAI Landing V3 sigue una arquitectura de Single Page Application (SPA) basada en Vue.js 3. La aplicación utiliza un patrón de diseño Component-Based Architecture que promueve la reutilización de código, mantenibilidad y escalabilidad.

3.2. Capas y Módulos

3.2.1. Capa de Presentación (Presentation Layer)

Esta capa maneja toda la interfaz de usuario y la experiencia visual:

  • App.vue: Componente raíz que orquesta la aplicación
  • Landing.vue: Componente principal que contiene toda la lógica de la landing page
  • Assets: Imágenes, iconos y recursos visuales

3.2.2. Capa de Lógica de Negocio (Business Logic Layer)

Maneja la lógica de la aplicación y el estado:

  • Countdown Timer: Lógica para calcular y mostrar el tiempo restante hasta el evento
  • Particle System: Configuración y gestión del sistema de partículas
  • Responsive Logic: Lógica para adaptar la interfaz a diferentes dispositivos

3.2.3. Capa de Servicios Externos (External Services Layer)

Integra librerías y servicios externos:

  • GSAP: Animaciones y transiciones
  • tsParticles: Efectos de partículas interactivas
  • Tailwind CSS: Sistema de estilos y diseño responsivo

3.3. Patrones de Diseño

3.3.1. Component-Based Architecture

La aplicación utiliza el patrón de componentes de Vue.js:

// Estructura de componentes
App.vue (Componente raíz)
└── Landing.vue (Componente principal)
    ├── Template (Vista)
    ├── Script (Lógica)
    └── Style (Estilos)

3.3.2. Composition API Pattern

Utiliza la Composition API de Vue 3 con <script setup>:

<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

// Reactive state
const countdown = ref({
  days: '00',
  hours: '00',
  minutes: '00',
  seconds: '00'
})

// Lifecycle hooks
onMounted(() => {
  // Setup logic
})

onUnmounted(() => {
  // Cleanup logic
})
</script>

3.3.3. Responsive Design Pattern

Implementa un patrón de diseño responsivo usando CSS Grid y Flexbox:

// Grid layout responsivo
.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 24px 80px;
}

@media (max-width: 768px) {
  .grid-container {
    grid-template-columns: 1fr;
  }
}

3.4. Flujo de Datos Principal

3.4.1. Inicialización de la Aplicación

1. main.js → Crea la aplicación Vue
2. App.vue → Monta el componente Landing
3. Landing.vue → Inicializa el estado y efectos
4. tsParticles → Configura el sistema de partículas
5. Countdown Timer → Inicia el contador regresivo

3.4.2. Flujo del Contador Regresivo

// Flujo de datos del countdown
1. Event DateTime (2025-09-18T18:00:00)
2. updateCountdown() → Calcula diferencia de tiempo
3. countdown.value → Actualiza estado reactivo
4. Template → Renderiza valores en tiempo real
5. setInterval() → Actualiza cada segundo

3.4.3. Flujo de Efectos Visuales

1. tsParticles → Inicializa sistema de partículas
2. GSAP → Configura animaciones
3. CSS Animations → Efectos de hover y transiciones
4. Responsive Breakpoints → Adapta layout a dispositivos

3.5. Gestión del Estado

3.5.1. Estado Reactivo

La aplicación utiliza el sistema de reactividad de Vue 3:

  • countdown: Estado del contador regresivo
  • timer: Referencia al intervalo del contador
  • backgroundTexture: Imagen de fondo
  • newLogoImg: Logo de la aplicación

3.5.2. Lifecycle Management

Gestión del ciclo de vida de componentes:

  • onMounted: Inicialización de efectos y timers
  • onUnmounted: Limpieza de recursos y timers
  • Cleanup: Prevención de memory leaks

3.6. Optimizaciones de Rendimiento

3.6.1. Lazy Loading

Los recursos se cargan de manera optimizada:

  • Imágenes optimizadas y comprimidas
  • CSS purgado automáticamente por Tailwind
  • JavaScript tree-shaken por Vite

3.6.2. Animaciones Optimizadas

Las animaciones utilizan técnicas de optimización:

  • CSS transforms para animaciones de GPU
  • requestAnimationFrame para animaciones suaves
  • Debouncing en eventos de resize

3.7. Escalabilidad

La arquitectura está diseñada para facilitar futuras expansiones:

  • Componentes Modulares: Fácil adición de nuevos componentes
  • Configuración Centralizada: Fácil modificación de parámetros
  • Separación de Responsabilidades: Lógica, presentación y estilos separados
  • API Ready: Preparado para integración con backend
