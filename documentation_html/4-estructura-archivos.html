4.1. Esquema de Directorios

La estructura del proyecto OlympicAI Landing V3 sigue las mejores prácticas para aplicaciones Vue.js con Vite:

olympicAI-landingV3/
├── .git/                    # Control de versiones Git
├── .vscode/                 # Configuración de Visual Studio Code
├── node_modules/            # Dependencias de Node.js (generado)
├── public/                  # Archivos públicos estáticos
│   └── vite.svg            # Icono de Vite
├── src/                     # Código fuente de la aplicación
│   ├── assets/             # Recursos estáticos (imágenes, iconos)
│   │   ├── img/           # Imágenes del proyecto
│   │   │   ├── asterisco.png
│   │   │   ├── background-bc.png
│   │   │   ├── background-no-texture.png
│   │   │   ├── background-texture.png
│   │   │   ├── baja-california.png
│   │   │   ├── golden-medal.png
│   │   │   ├── logo-blanco.png
│   │   │   ├── logo-blanco-sm.png
│   │   │   ├── logo-negro.png
│   │   │   ├── logo-negro-sm.png
│   │   │   └── newLogo.png
│   │   └── vue.svg        # Icono de Vue.js
│   ├── components/         # Componentes Vue.js
│   │   └── Landing.vue    # Componente principal de la landing page
│   ├── App.vue            # Componente raíz de la aplicación
│   ├── main.js            # Punto de entrada de la aplicación
│   └── style.css          # Estilos globales
├── .gitignore             # Archivos ignorados por Git
├── index.html             # Archivo HTML principal
├── package-lock.json      # Lock file de dependencias
├── package.json           # Configuración del proyecto y dependencias
├── postcss.config.js      # Configuración de PostCSS
├── README.md              # Documentación del proyecto
├── tailwind.config.js     # Configuración de Tailwind CSS
└── vite.config.js         # Configuración de Vite

4.2. Archivos Clave

4.2.1. Archivos de Configuración

package.json

Archivo principal de configuración del proyecto que define:

  • Nombre del proyecto: olympicai-landingv2
  • Versión: 0.0.0
  • Tipo: Módulo ES
  • Scripts: dev, build, preview
  • Dependencias: Vue 3, GSAP, tsParticles, Lucide Vue
  • DevDependencies: Vite, Tailwind CSS, PostCSS, Autoprefixer

vite.config.js

Configuración del bundler Vite:

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
})

tailwind.config.js

Configuración del framework CSS Tailwind:

/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{vue,js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

postcss.config.js

Configuración de PostCSS para procesamiento de CSS:

export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}

4.2.2. Archivos de Entrada

index.html

Archivo HTML principal que sirve como punto de entrada:

  • Define la estructura HTML base
  • Incluye el div con id "app" para montar Vue
  • Configura el viewport para responsividad
  • Establece el título de la página

src/main.js

Punto de entrada de JavaScript que inicializa la aplicación Vue:

import { createApp } from 'vue'
import './style.css'
import App from './App.vue'

createApp(App).mount('#app')

4.2.3. Componentes Vue

src/App.vue

Componente raíz de la aplicación:

  • Utiliza Composition API con <script setup>
  • Importa y renderiza el componente Landing
  • Define estilos scoped para el componente

src/components/Landing.vue

Componente principal que contiene toda la lógica de la landing page:

  • Template: Estructura HTML con diseño responsivo
  • Script: Lógica del contador regresivo y efectos visuales
  • Style: Estilos CSS personalizados y animaciones

4.2.4. Recursos Estáticos

src/assets/img/

Directorio que contiene todas las imágenes del proyecto:

  • newLogo.png: Logo principal de las olimpiadas
  • background-*.png: Imágenes de fondo con diferentes texturas
  • logo-*.png: Variantes del logo (blanco/negro, tamaño pequeño/grande)
  • golden-medal.png: Imagen de medalla dorada
  • baja-california.png: Imagen relacionada con la ubicación
  • asterisco.png: Elemento decorativo

4.2.5. Estilos

src/style.css

Archivo de estilos globales que incluye:

  • Importación de Tailwind CSS
  • Estilos base personalizados
  • Configuración de fuentes y colores

4.3. Organización de Archivos

4.3.1. Separación de Responsabilidades

La estructura sigue el principio de separación de responsabilidades:

  • Configuración: Archivos de configuración en la raíz
  • Código Fuente: Todo el código en el directorio src/
  • Componentes: Componentes Vue en src/components/
  • Recursos: Imágenes y assets en src/assets/
  • Público: Archivos estáticos en public/

4.3.2. Convenciones de Nomenclatura

El proyecto sigue convenciones estándar:

  • Componentes Vue: PascalCase (Landing.vue)
  • Archivos JavaScript: camelCase (main.js)
  • Archivos de Configuración: kebab-case (vite.config.js)
  • Imágenes: kebab-case (background-texture.png)

4.4. Archivos de Desarrollo

4.4.1. .gitignore

Define qué archivos y directorios ignorar en el control de versiones:

  • node_modules/
  • dist/
  • .env.local
  • Archivos de sistema operativo

4.4.2. .vscode/

Configuración específica para Visual Studio Code que puede incluir:

  • Configuración de extensiones
  • Configuración de debugging
  • Configuración de formateo de código

4.5. Archivos Generados

4.5.1. node_modules/

Directorio generado automáticamente que contiene:

  • Todas las dependencias instaladas via npm
  • Dependencias transitivas
  • Archivos de distribución de librerías

4.5.2. package-lock.json

Archivo generado automáticamente que:

  • Bloquea las versiones exactas de todas las dependencias
  • Garantiza builds reproducibles
  • Optimiza la instalación de dependencias

4.6. Estructura de Build

Cuando se ejecuta el comando de build, Vite genera:

  • dist/: Directorio con archivos optimizados para producción
  • Assets optimizados: CSS y JS minificados
  • Imágenes optimizadas: Comprimidas y con nombres hash
