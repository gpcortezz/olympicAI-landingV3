5.1. Requisitos del Sistema

Para ejecutar el proyecto OlympicAI Landing V3, se requieren los siguientes componentes:

5.1.1. Requisitos de Software

  • Node.js: Versión 16.0.0 o superior
  • npm: Versión 8.0.0 o superior (incluido con Node.js)
  • Navegador Web: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
  • Sistema Operativo: Windows 10+, macOS 10.15+, Linux (Ubuntu 18.04+)

5.1.2. Requisitos de Hardware

  • RAM: Mínimo 4GB, recomendado 8GB
  • Almacenamiento: Mínimo 1GB de espacio libre
  • Procesador: Cualquier procesador moderno de 2GHz o superior
  • Conectividad: Conexión a internet para descargar dependencias

5.2. Variables de Entorno

Este proyecto es una aplicación frontend que no requiere variables de entorno específicas para su funcionamiento básico. Sin embargo, para futuras integraciones, se pueden considerar las siguientes variables:

Variable Descripción Ejemplo Obligatorio
NODE_ENV Entorno de ejecución de Node.js development | production No
VITE_APP_TITLE Título de la aplicación OlympicAI Landing No
VITE_EVENT_DATE Fecha del evento (ISO string) 2025-09-18T18:00:00 No

5.3. Pasos de Instalación (Entorno de Desarrollo)

5.3.1. Clonar el Repositorio

# Clonar el repositorio
git clone [URL_DEL_REPOSITORIO]
cd olympicAI-landingV3

5.3.2. Instalar Dependencias

# Instalar todas las dependencias del proyecto
npm install

5.3.3. Ejecutar en Modo Desarrollo

# Iniciar el servidor de desarrollo
npm run dev

El servidor de desarrollo se iniciará en http://localhost:5173 (o el puerto disponible más cercano).

5.3.4. Verificar la Instalación

Una vez que el servidor esté ejecutándose, deberías ver:

  • La landing page con el diseño futurista
  • El contador regresivo funcionando
  • Los efectos de partículas en el fondo
  • Diseño responsivo en diferentes tamaños de pantalla

5.5. Pasos de Despliegue (Entorno de Producción)

5.5.1. Build para Producción

# Construir la aplicación optimizada
npm run build

Este comando generará un directorio dist/ con los archivos optimizados para producción.

5.5.2. Estructura de Build de Producción

El directorio dist/ contendrá:

  • index.html: Archivo HTML principal optimizado
  • assets/: Directorio con archivos CSS y JS minificados
  • Imágenes optimizadas: Comprimidas y con nombres hash

5.5.3. Opciones de Despliegue

Despliegue en Servidor Web Estático

# Copiar archivos del directorio dist/ al servidor web
# Ejemplo para Apache/Nginx
cp -r dist/* /var/www/html/

# O para un servidor local de prueba
npx serve dist/

Despliegue en CDN/Cloud Storage

  • Netlify: Drag & drop del directorio dist/
  • Vercel: Integración con Git automática
  • AWS S3: Subir archivos al bucket
  • GitHub Pages: Configurar GitHub Actions

Despliegue con Docker

Crear un Dockerfile para containerización:

# Dockerfile
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

5.6. Configuración de Servidores Web

5.6.1. Configuración Nginx

server {
    listen 80;
    server_name tu-dominio.com;
    root /var/www/html;
    index index.html;

    # Configuración para SPA
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache para assets estáticos
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

5.6.2. Configuración Apache

<VirtualHost *:80>
    ServerName tu-dominio.com
    DocumentRoot /var/www/html
    
    <Directory /var/www/html>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    
    # Configuración para SPA
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^(.*)$ /index.html [L]
</VirtualHost>

5.7. Optimizaciones de Producción

5.7.1. Optimizaciones Automáticas de Vite

Vite aplica automáticamente las siguientes optimizaciones:

  • Tree Shaking: Elimina código no utilizado
  • Minificación: Comprime CSS y JavaScript
  • Code Splitting: Divide el código en chunks
  • Asset Optimization: Optimiza imágenes y otros assets

5.7.2. Configuraciones Adicionales Recomendadas

  • Compresión Gzip: Habilitar en el servidor web
  • HTTP/2: Habilitar para mejor rendimiento
  • CDN: Usar CDN para assets estáticos
  • Cache Headers: Configurar headers de cache apropiados
